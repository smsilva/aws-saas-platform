## Traffic flow

```
Internet → ALB (TLS termination, ACM cert)
         → WAF (managed rules + rate limiting)
         → Istio IngressGateway (ClusterIP, private subnet)
         → Istio Gateway + VirtualService
         → Application (namespace with sidecar injection)
```

## Rule: Visual design must always be respected

- Any change made in `lab/aws/eks/design` must be reflected in the frontend `lab/aws/eks/services`, and vice-versa
- Changes must be applied by creating an implementation plan, which must be approved before execution
- **Mandatory check:** after any visual change, verify both `make serve` (sandbox) and the real FastAPI service (`uvicorn`) to ensure parity

### Sync architecture (Design ↔ Services)

- **CSS:** `design/index.html` loads `services/tenant-frontend/app/static/app.css` directly via `<link>`. Editing `app.css` syncs sandbox and service automatically. The inline `<style>` in `design/index.html` contains only sandbox chrome (`#sandbox-bar`) and minor overrides.
- **Shared CSS:** `app/static/shared/` is a symlink to `../../../../design/shared`. Build scripts (`06-deploy-services`, `13-deploy-services`) replace the symlink with a real copy before `docker build` and restore it after.
- **UI JS:** extracted to `services/tenant-frontend/app/static/test-ui.js`. Interface: `window.initTestPage({ testCases, jwtToken })`. Both `design/index.html` and `test.html` call this function — editing the file syncs both.
- **External dependencies (CDN, JS/CSS libraries):** any `<script>` or `<link>` from a CDN added to `test.html` (service) **must also be added** to `design/index.html` (sandbox), and vice-versa. Example: highlight.js is declared in both. Omitting on one side silently breaks visual parity.

## Rule: always add a destroy entry when creating a new resource using scripts

Every time a provisioning script creates an AWS or Azure resource, **the corresponding deletion entry must be added to `scripts/destroy` in the same session**, at the correct position in reverse order. The order must respect dependencies (e.g., Cognito custom domain before the User Pool) — if there is a conflict with cost order, dependency order takes precedence.

Checklist when adding a resource:
1. Identify the deletion command (`aws <service> delete-*` or `az ... delete`)
2. Determine the correct position in `destroy` (reverse creation order, respecting dependencies)
3. Use `|| true` or an existence check to make it idempotent
4. Update the order comment in the `destroy` header
5. Update HANDOFF.md if there was a pending entry in the backlog

## Before creating or destroying resources

Always run `bootstrap` first to validate prerequisites:

```bash
# before creating
./scripts/bootstrap --create

# before destroying
./scripts/bootstrap --destroy
```

The script checks: installed CLIs (`aws`, `eksctl`, `kubectl`, `helm`, `istioctl`, `docker`, `az`), active AWS credentials on the correct account (`221047292361`), Azure CLI authenticated on subscription `wasp-sandbox`, Docker daemon running, and mandatory secrets defined (`GOOGLE_CLIENT_ID`, `GOOGLE_CLIENT_SECRET`, `STATE_JWT_SECRET`).

## Operational gotchas

### `tenants.json` must have real Cognito values

`services/discovery/app/data/tenants.json` is a static data source. When reprovisioning Cognito, update `client_id` and `idp_pool_id` before the build, commit, and rebuild with a new SHA tag.

### `COGNITO_DOMAIN` without `https://`

In the `platform-frontend-config` ConfigMap, the `COGNITO_DOMAIN` field must be the hostname only (`idp.wasp.silvios.me`). The code in `auth.py` already adds the scheme — providing the full URL produces `https://https://idp...`.

### DynamoDB — reserved words in `--update-expression`

Attributes with reserved names (e.g., `auth`, `name`, `status`) cause a `ValidationException`. Use `--expression-attribute-names` with a `#` alias:

```bash
--update-expression 'SET #auth.field = :val' \
--expression-attribute-names '{"#auth": "auth"}'
```

### WAFv2 — `--id` requires a UUID, not a name

```bash
# CORRECT — $NF extracts the UUID (last segment of the ARN)
web_acl_id="$(echo "${web_acl_arn}" | awk -F'/' '{print $NF}')"
```

### Pipe + Python heredoc — stdin conflict

A pipe (`|`) and a heredoc (`<<EOF`) compete for stdin. The heredoc wins. Write the variable to a temp file and read it via `open()`.

### Script 17 does not rebuild `discovery`

`17-deploy-customer2` only builds/pushes `platform-frontend` and `callback-handler`. If the `discovery` service is modified, re-run `13-deploy-services` to rebuild and redeploy it.
