# SEC-005 — No dedicated Security Groups for the ALB

| Field | Value |
|---|---|
| **Severity** | Low |
| **Script** | `scripts/07-configure-alb-ingress` |
| **Status** | Open |

## Problem

The Ingress does not define the `alb.ingress.kubernetes.io/security-groups` annotation, leaving the ALB Controller to create and manage Security Groups automatically with permissive default rules (0.0.0.0/0 on ports 80 and 443).

## Impact

In production, this prevents explicit control of which IPs can access the ALB — making it impossible to restrict by source IP (e.g., corporate IPs only) or to integrate with prefix lists.

## Fix

For production, create and reference explicit Security Groups:

```yaml
alb.ingress.kubernetes.io/security-groups: sg-xxxx,sg-yyyy
alb.ingress.kubernetes.io/manage-backend-security-group-rules: "true"
```

For the lab, the default behavior is acceptable since the WAF filters traffic at layer 7.
