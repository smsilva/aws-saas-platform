# SEC-002 — IAM policy download without integrity verification

| Field | Value |
|---|---|
| **Severity** | Medium |
| **Script** | `scripts/04-install-alb-controller`, line 19 |
| **Status** | Open |

## Problem

The IAM policy is downloaded directly from GitHub with `curl -sL` and immediately applied to the AWS account without hash or signature verification:

```bash
curl -sL \
  "https://raw.githubusercontent.com/kubernetes-sigs/aws-load-balancer-controller/${alb_controller_version}/docs/install/iam_policy.json" \
  -o /tmp/alb-iam-policy.json
```

## Impact

If the file content is modified (upstream repository compromise, MITM attack, or use of a tampered version), the resulting IAM policy may grant excessive permissions on the AWS account.

## Fix

Verify the SHA256 hash of the file after download, comparing against the value published in the official release changelog:

```bash
curl -sL "https://raw.githubusercontent.com/kubernetes-sigs/aws-load-balancer-controller/${alb_controller_version}/docs/install/iam_policy.json" \
  -o /tmp/alb-iam-policy.json

# Verify hash (get expected value from release notes)
echo "<expected_sha256>  /tmp/alb-iam-policy.json" | sha256sum -c
```
