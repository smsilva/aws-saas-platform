# SEC-007 — WAF without rate limiting

| Field | Value |
|---|---|
| **Severity** | Low |
| **Script** | `scripts/09-configure-waf` |
| **Status** | Resolved by `scripts/15-configure-waf-ratelimit` |

## Problem

The WebACL uses only managed pattern detection rules (CRS, KnownBadInputs, IP Reputation), but does not configure rate limiting rules. Shield Standard does not protect against high-volume application-layer attacks.

## Impact

The application is vulnerable to brute-force attacks on authentication endpoints or request flooding from a single IP.

## Fix

Add a rate limit rule to the WebACL:

```json
{
  "Name": "RateLimitRule",
  "Priority": 0,
  "Action": {"Block": {}},
  "Statement": {
    "RateBasedStatement": {
      "Limit": 2000,
      "AggregateKeyType": "IP"
    }
  },
  "VisibilityConfig": {
    "SampledRequestsEnabled": true,
    "CloudWatchMetricsEnabled": true,
    "MetricName": "RateLimitRule"
  }
}
```

## Implementation

The `scripts/15-configure-waf-ratelimit` script implements rate limiting specific to `/login` and `/callback` (100 req/5min per IP), addressing this item for authentication endpoints.
