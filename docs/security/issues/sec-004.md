# SEC-004 — Admin permission granted without namespace scope

| Field | Value |
|---|---|
| **Severity** | Medium |
| **Script** | `scripts/03-configure-access` |
| **Status** | Open |

## Problem

The caller IAM receives the `AmazonEKSClusterAdminPolicy` policy with `access-scope type=cluster`, which is equivalent to `cluster-admin` in Kubernetes RBAC — unrestricted access to all resources and namespaces:

```bash
aws eks associate-access-policy \
  --policy-arn arn:aws:eks::aws:cluster-access-policy/AmazonEKSClusterAdminPolicy \
  --access-scope type=cluster
```

## Impact

If the caller's IAM role is compromised, the attacker has full control of the cluster.

## Fix

For operational use, prefer `AmazonEKSAdminPolicy` with `access-scope type=namespace` restricted to the required namespaces. For the lab, document that this access level is intentional only for the lab environment.
